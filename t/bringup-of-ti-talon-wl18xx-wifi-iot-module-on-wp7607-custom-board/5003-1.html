thanks Scott and jyijyi for the responses.

Yes, the Mirage Wl18xx card uses 1.8v for SDIO lines.

In our custom board, we had connected the MDM SDIO lines, which is 2.85 v directly to the Mirage SDIO lines. This broke the mirage module.
Was able to get the MDM SDIO lines to 1.8v after changing in the DTSI file.

In the MangoH red board, there is a level translator before connecting SDIO to the Mirage.

So currently in our custom board, we have added a level translator.

Thanks for the responses.

tejaswisj August 26, 2020, 9:56am 23
  1. arch/arm/boot/dts/qcom/mdm9607-regulator.dtsi
    change the regulator l13 init-voltage from 2.85 to 1.8v
    regulator-name = “mdm9607_l13”;
    regulator-min-microvolt = <1800000>;
    regulator-max-microvolt = <2850000>;
  •                   qcom,init-voltage = <2850000>;

  •                   qcom,init-voltage = <1800000>;
                  status = "okay";
          };
  };
  1. in /arch/arm/boot/dts/qcom/mdm9607-wp76xx.dtsi
    Change the voltage levels.

&sdhc_2 {
vdd-supply = <&sdcard_ext_vreg>;

  •   qcom,vdd-voltage-level = <2850000 2850000>;

  •   qcom,vdd-voltage-level = <1800000 1800000>;
  qcom,vdd-current-level = <15000 400000>;

  vdd-io-supply = <&mdm9607_l13>;

  •   qcom,vdd-io-voltage-level = <1800000 2850000>;

  •   qcom,vdd-io-voltage-level = <1800000 1800000>;
  qcom,vdd-io-current-level = <200 50000>;

  #address-cells = <0>;
